# 다드림 청년부 대시보드 — 설계 문서

- **날짜**: 2026-06-10
- **대상**: 제자광성교회 청년부(다드림) 내부 운영 대시보드
- **사용자**: 목사님, 부장·부감 집사님, 임원, 일반 청년 (명부 기준 150~300명)

## 1. 목적

청년부 운영(심방, 명부, 예배위원, 임원회의)을 한 곳에서 관리하는 비공개 웹 대시보드.
오랫동안 유지보수 부담 없이, 무료에 가까운 비용으로 운영하는 것이 핵심 목표.

## 2. 기술 스택

| 영역 | 선택 | 비고 |
|------|------|------|
| 프레임워크 | Next.js (App Router) + TypeScript | 화면 + API 단일 프로젝트 |
| DB / 인증 / 스토리지 | Supabase | PostgreSQL + RLS, Kakao OAuth, 비공개 Storage |
| UI | Tailwind CSS + shadcn/ui | 모바일 우선 반응형 |
| 배포 | Vercel | git push 자동 배포, 무료 플랜 |

## 3. 인증 및 권한

### 3.1 로그인
- **카카오 로그인 + 승인제**. Supabase Auth의 Kakao OAuth 사용.
- 가입 직후 `승인 대기(pending)` 상태 — 대시보드 접근 불가, 안내 화면만 표시.

### 3.2 역할 4단계
| 역할 | 코드 | 권한 요약 |
|------|------|----------|
| 목사님 | `pastor` | 전체 + 심방 확정 + 목양 메모 |
| 부장·부감 | `staff` | 전체 열람 (목양 메모 포함 여부는 목사님 전용으로 제한) |
| 임원 | `officer` | 가입 승인, 명부 관리, 체크리스트·예배위원·결혼 리스트 관리, 연락처 열람 |
| 청년 | `member` | 본인 정보 + 명부 기본 정보 열람, 심방 신청 |

### 3.3 명부-계정 분리 및 연결
- `members`(명부)와 로그인 계정(`auth.users` + `profiles`)을 분리.
- 명부는 계정 없이도 전원 등록 (임원 직접 입력 또는 엑셀 일괄 등록).
- 가입 승인 시 카카오 프로필 이름으로 명부 후보를 자동 추천 → 임원이 확인 후 연결.
- 동명이인은 후보 복수 표시, 명부에 없는 새가족은 승인과 동시에 명부 신규 생성.
- 연결 즉시 기존 데이터(셀 소속, 심방 이력, 예배위원 이력)가 계정과 연동.

## 4. 페이지 구성

### 4.1 홈 대시보드
- 이번 주 / 다음 주 예배위원
- 다가오는 청년부 일정 (7일)
- 내 심방 신청 현황 (역할별: 목사님은 대기 중 신청 목록)
- 앱 내 알림 배지

### 4.2 심방 캘린더
- 월간 캘린더에 청년부 일정 + 심방 일정 통합 표시.
- **신청 흐름 (자유 신청 → 목사님 확정)**:
  1. 청년이 희망 날짜·시간대(복수 선택 가능)와 나눔 내용을 적어 신청
  2. 목사님이 `확정` / `다른 시간 제안` / `반려`
  3. 시간 제안 시 청년이 수락하면 확정
- 상태: `신청(requested)` → `조정 중(proposed)` → `확정(confirmed)` → `완료(completed)` / `반려(declined)` / `취소(cancelled)`
- 계정 없는 청년의 심방은 임원이 대신 신청 입력 가능.
- 심방 완료 후 **목양 메모**: 목사님만 작성·열람.
- 심방 일정의 캘린더 노출: 본인 + 목사님 + 부장·부감에게만 상세 표시, 그 외에는 비공개.

### 4.3 청년 명부
- 사진 카드 그리드, 이름/셀 검색·필터. 300명 대응 (페이지네이션 또는 무한 스크롤).
- 상세 페이지 **권한별 차등 열람**:
  - `member`: 이름, 사진, 셀, 임원 여부
  - `officer`: + 연락처, 생년월일, 직분, 세례 여부
  - `staff`/`pastor`: + 주소, 직장, 가족 관계, 심방 이력, 출석 현황
- 본인은 본인 정보 전체 열람 및 일부 수정(연락처 등) 가능.

### 4.4 결혼 예정 리스트
- 명부와 연동. 예식일, 장소, 배우자(명부 내/외부), 비고.
- D-day순 정렬. `officer` 이상 열람·관리.

### 4.5 임원회의 체크리스트
- 회의 단위로 안건·할 일 목록 작성, 담당자 지정, 완료 체크.
- 미완료 항목은 다음 회의로 이월.
- `officer` 이상 열람·관리.

### 4.6 예배위원 배정
- 주차별 배정표. 직책은 관리 화면에서 추가/수정 가능 (사회, 대표기도, 찬양인도, 헌금, 안내 등 — 초기값은 실제 다드림 직책 목록으로 시드).
- 명부와 연동하여 배정. 이번 주/다음 주는 홈에 노출.
- 전체 열람, `officer` 이상 편집.

## 5. 데이터 모델 (Supabase PostgreSQL)

```
profiles            -- auth.users 1:1, 역할(role), 승인 상태, member_id 연결
members             -- 명부 기본: 이름, 사진 경로, 생년월일, 연락처, 셀, 직분, 세례, 임원 여부, 재적 상태
member_private      -- 민감 정보 분리: 주소, 직장, 가족 관계 (staff/pastor + 본인만)
cells               -- 셀/조직 구조
visit_requests      -- 심방: 신청자(member_id), 희망 일시(복수), 사연, 상태, 확정 일시
visit_notes         -- 목양 메모 (pastor 전용)
events              -- 청년부 일정
wedding_plans       -- 결혼 예정: member_id, 예식일, 장소, 배우자
meetings            -- 임원회의
meeting_items       -- 안건/할 일: 담당자, 완료 여부, 이월 출처
service_roles       -- 예배위원 직책 (커스텀 가능)
service_assignments -- 주차별 직책-청년 배정
notifications       -- 앱 내 알림
```

### 보안 원칙
- **RLS(Row Level Security)로 모든 테이블의 열람·수정 권한을 DB 차원에서 강제** — 화면에서 가리는 것이 아니라 DB가 거부.
- 민감 정보는 `member_private` 테이블로 물리적 분리.
- 사진은 비공개 Storage 버킷 + signed URL (로그인 사용자에게만 발급).
- `pending` 상태 계정은 모든 데이터 접근 불가.

## 6. 알림

- 1차: **앱 내 알림만** (`notifications` 테이블 + 헤더 배지).
- 대상 이벤트: 심방 신청(→목사님), 확정/제안/반려(→신청자), 가입 승인(→본인), 예배위원 배정(→해당자).
- 카카오 알림톡은 2차 확장으로 보류 (사업자 등록·비용 필요).

## 7. 비기능 요구사항

- **모바일 우선** 반응형. 임원·목사님용 관리 화면은 PC 레이아웃 최적화 병행.
- 무료 플랜 운영: Supabase Free + Vercel Hobby로 시작.
- 한국어 UI, Asia/Seoul 시간대 고정.

## 8. 개발 순서

1. **1차 (MVP)**: 프로젝트 셋업, 카카오 로그인 + 승인/명부 연결, 청년 명부(권한별 열람), 심방 캘린더
2. **2차**: 예배위원 배정, 임원회의 체크리스트, 결혼 예정 리스트, 홈 대시보드 완성
3. **3차 (확장)**: 카카오 알림톡, 출석 관리 고도화 등

## 9. 사용자 준비 사항

- 카카오 개발자 앱 등록 (로그인용 — 단계별 안내 예정)
- 명부 초기 데이터 (엑셀 등 보유 형태 그대로)
- 셀/조직 구조, 예배위원 직책 목록
- 도메인 (없으면 `*.vercel.app`으로 시작)
